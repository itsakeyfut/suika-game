# スイカゲーム - 実装ロードマップ

## 概要

このディレクトリには、スイカゲームプロジェクトの実装ロードマップが含まれています。各フェーズは独立したロードマップファイルとして管理され、GitHub Issueとして起票できるレベルの詳細なタスクリストを含んでいます。

## ロードマップの使い方

1. **現在のフェーズを確認**: 下記の進行状況表を参照
2. **フェーズのロードマップを開く**: 該当するphase-XX.mdを開く
3. **タスクをIssue化**: 各タスクをGitHub Issueとして起票
4. **実装**: タスクの受け入れ基準を満たすまで実装
5. **完了チェック**: 各タスクの完了条件を確認
6. **次のフェーズへ**: フェーズの全タスク完了後、次フェーズに進む

## 凡例

- ✅ **完了**: フェーズの全タスクが完了し、検証済み
- 🚀 **進行中**: フェーズのタスクを実装中
- 🔲 **未着手**: まだ開始していない
- ⏸️ **保留**: 何らかの理由で一時停止中
- ❌ **ブロック**: 依存関係により開始できない

## GitHub Issue起票ガイド

### タスクのIssue化方法

各フェーズのロードマップには、以下の形式でタスクが記載されています：

```markdown
#### タスク X.Y: タスク名

**優先度**: P0/P1/P2/P3
**推定工数**: X-Y時間
**ラベル**: task, phase-X

**説明**:
タスクの詳細説明

**受け入れ基準**:
- [ ] AC1: 条件1
- [ ] AC2: 条件2

**実装ガイド**:
実装の手順や注意点
```

このタスクをGitHub Issueとして起票する際は、以下のテンプレートを使用してください：

**Issueタイトル**:
```
[Phase X] タスク名
```

**Issueボディ**:
```markdown
## タスク概要
[タスクの説明をコピー]

## 受け入れ基準
- [ ] AC1: 条件1
- [ ] AC2: 条件2

## 実装ガイド
[実装ガイドをコピー]

## 関連ドキュメント
- [該当するフェーズのロードマップへのリンク]
- [関連する設計ドキュメントへのリンク]

## チェックリスト
- [ ] 実装完了
- [ ] ユニットテスト追加（該当する場合）
- [ ] ゲーム内で動作確認
- [ ] `just check` 実行済み
- [ ] ドキュメント更新（必要な場合）
```

**ラベル設定**:
- `task` または `enhancement`
- `phase-X` (例: `phase-2`)
- 優先度ラベル: `P0`, `P1`, `P2`, `P3`

### タスクの優先度

- **P0 (Critical)**: ゲームの基本機能に必須。最優先で実装
- **P1 (High)**: 重要な機能。早めに実装すべき
- **P2 (Medium)**: あると良い機能。時間があれば実装
- **P3 (Low)**: 追加機能や改善。後回しでも可

## 進行管理のヒント

### 1. 1フェーズずつ着実に進める
- 複数フェーズを同時並行しない
- 各フェーズの完了条件をすべて満たしてから次に進む

### 2. タスクの粒度を適切に保つ
- 1タスクは1-3時間程度の作業量に収める
- 大きすぎるタスクは分割する
- 小さすぎるタスクは統合する

### 3. 定期的に検証する
- 各タスク完了時に動作確認
- フェーズ完了時に総合テスト
- 問題があればすぐに修正

### 4. ドキュメントを更新する
- 実装と異なる仕様が出た場合、ドキュメントを更新
- 新しい知見があれば追記
- コード内コメントも充実させる

### 5. コミット戦略
- 各タスク完了時にコミット
- コミットメッセージ: `feat(scope): タスク名`
- フェーズ完了時にPRを作成（推奨）
